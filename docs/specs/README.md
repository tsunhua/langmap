## 文档说明

### project-spec.md
LangMap 项目的粗略需求规范，包括：
- 项目目标：构建语言地图网站，显示各语言丰富度差异
- 语料来源（三种）
  1. 权威书籍、影视、音乐作品
  2. AI 补充（有 AI 标识，提供赞和踩按钮）
  3. 用户提交（需 AI 审核，提供赞和踩按钮）
- 页面结构
  1. 首页：创造性地显示各语言丰富度差异
  2. 查询词条页面：支持跨语言查询，跳转外部软件朗读，命中语料库可进入语言地图
  3. 语言地图：查询某一词/句在各语言的表达
- 语言细化：支持到地区级别 `[语言]-[国家城市][乡镇]`，通过地理位置选择器选择，最多截取到乡镇
- 粗略架构
  - 前端：Vue 3
  - 后端：Python 3.12

## 核心需求

### 功能需求

1. **语言展示**
   - 支持多种语言
   - 显示语言丰富度差异
   - 支持地域化到乡镇级别

2. **查询功能**
   - 支持跨语言查询
   - 未命中时提供 AI 补全选项
   - 支持用户提交补全
   - 支持外部朗读跳转

3. **语料来源**
   - 权威来源（书籍、影视、音乐）
   - AI 生成内容（需标注）
   - 用户提交（需审核）
   - 所有来源提供赞/踩功能

4. **质量控制**
   - AI 内容自动审核
   - 用户提交审核队列
   - 信任评分系统
   - 版本历史记录

### 非功能需求

1. **性能**
   - 响应时间 < 500ms
   - 支持并发查询
   - 数据库优化

2. **可用性**
   - 99.5% 以上正常运行时间
   - 友好的用户界面
   - 多语言支持

3. **安全性**
   - 用户认证与授权
   - HTTPS 加密
   - 防 SQL 注入
   - 防 XSS 攻击

4. **可扩展性**
   - 支持数据增长
   - 支持语言扩展
   - 模块化架构

## 市场定位

LangMap 的核心差异化：
- **细粒度地域化**：细化到乡镇级别，多数竞品仅到国家/城市
- **来源可视化**：强制标注来源（authoritative/AI/user），UI 明显显示
- **AI 辅助但透明**：AI 补全明显标识，用户可评分
- **外部朗读集成**：支持跳转外部朗读服务（Forvo、系统 TTS 等）
- **审核机制**：用户提交需审核，保证质量

目标市场：
- **主要市场**：语言研究机构、文化保护组织
- **次要市场**：语言学习者、翻译从业者、AI 公司
