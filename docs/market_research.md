# 市场调研：语言地图产品可行性与定位

## 目标
- 为 `langmap` 项目提供简明的市场与竞品分析、目标用户画像、商业化路径建议，以及差异化定位。

## 问题陈述
需要一个网站来展示“各语言丰富度的差异”，支持三类语料来源（权威、AI 补充、用户提交），并在地域上细化到乡镇级别，提供查询与可视化语言地图。

## 主要竞品与参考项目
- **WALS (World Atlas of Language Structures)**: 学术型语言特征资料库，侧重结构特征与地理分布。
- **Ethnologue / Glottolog**: 语言分类与元数据，是权威语言列表与数量统计来源。
- **Omniglot**: 语言字母与书写样例，面向大众的语言资源汇总。
- **Forvo**: 社区驱动的发音语料库（语音优先），用户贡献与质量控制值得参考。
- **Tatoeba**: 可搜索的例句数据库，社区贡献+开源许可，适合多语言例句汇聚。
- **Wiktionary / Wikidata**: 开放词典与结构化数据，可作为权威与补充数据源。
- **Google Translate / DeepL**: 强大的翻译 & 发音工具，作为外部朗读与快速补全的参考，但非可直接复用数据源。

竞品短评：现有项目要么学术性强但不够可交互（WALS），要么是社区驱动但缺少严格来源标注（Tatoeba）。`langmap` 可通过“细粒度地域化 + 来源可视化 + AI 辅助补全”和“可听化/朗读跳转”形成差异化。

## 目标用户群
- **语言学研究者与田野工作者**：需要地域化、可溯源的实例与例句。
- **语言学习者与文化爱好者**：希望听到地道发音与常用表达。
- **翻译 / 本地化工程师**：需要多语表达对照与地域变体信息。
- **文化机构 / 教育组织**：保存濒危语言样本，展示语言多样性。
- **AI/数据公司**：需要高质量、多样化的语言语料来训练或评估系统（对数据开放或付费 API 有需求）。

## 用户痛点 & 价值主张
- 痛点：不同语言/方言的表达分散、难以按地域细化、来源不透明、缺乏音频/朗读链接。
- 价值：统一语料浏览与地图可视化、来源与可信度显式标注、支持 AI 辅助补全但明显标识、用户贡献 + 审核机制保证可持续增长。

## 商业化与可持续模式（备选）
- **捐赠 / 非营利+研究资助**：适合学术导向或公共文化项目。
- **数据授权 / API 付费**：向企业、研究机构提供结构化数据/音频访问。
- **SaaS/付费功能**：高级可视化、批量导出、私有数据集支持。
- **会员/社区支持**：付费用户可获得高级审核优先权或贡献奖励。

## 法律与伦理考虑
- 用户提交与媒体摘录需注意版权（尤其是影视/音乐）；建议优先使用可授权/引用的资料或做摘要化表示并保留来源。
- 敏感或受保护群体语料需谨慎处理，可能需要额外的同意或脱敏策略。

## 市场机会与风险
- 机会：少有产品同时满足“地域细粒度 + 来源可视化 + 可听化”三项需求。对濒危语言保护、文化机构有较强吸引力。
- 风险：数据收集和审核成本高；音频与版权处理带来持续成本；若过度依赖第三方 AI 补全，可能引发准确性和信任问题。

## 推荐差异化策略（快速列表）
- 强制来源标签：`authoritative / AI / user`，并在 UI 明显显示。
- 地域选择器：以地图/位置选择为主，默认使用用户当前位置，限制最细粒度为乡镇。
- AI 补全部分必须打上 AI 标识，允许用户“赞/踩”并进入人工审核队列。
- 提供外部朗读跳转（例如：系统 TTS、Forvo 链接或浏览器播放）而非仅依赖单一服务。

## 建议的 MVP 特性（最小可行产品）
1. 基本数据模型：语言、地域层级、短句/词条、来源标签、可播放音频 URL、上下文说明。
2. 搜索页面：按语言与地域检索表达，未命中时展示 AI 补全按钮与用户提交按钮。
3. 地图视图：可查询并在地图上展示某一词/句在不同地域的表达（气泡或热力）。
4. 简单审核队列：AI 先筛，人工确认，展示来源与审核状态。
5. 基本用户交互：赞/踩、提交历史、条目详情带来源。

## 结论（市场可行性）
产品有明显的利基市场——面向研究机构、文化保护与高阶语言学习/本地化用户。通过明确的来源标签与审查流程，可以在信任层面建立优势，商业化路径以数据授权与机构合作为主。
