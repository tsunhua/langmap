# 详情页智能搜索与关联设计

## 概述

优化详情页的"搜索和关联"功能，简化用户操作流程，提升用户体验。主要解决两个问题：
1. 当前交互流程繁琐（需要多次点击、手动触发搜索）
2. 相同书写系统的语言间搜索干扰（如简体中文/繁体中文）

## 当前问题分析

### 现有交互流程

1. 用户点击"搜索和关联"标签页或"关联词句"按钮
2. 进入搜索模式，需要手动点击启用搜索
3. 输入查询词
4. 点击"搜索"按钮
5. 查看搜索结果
6. 如果没找到结果，点击"添加词句"按钮

### 存在的问题

1. **操作繁琐**
   - 需要多次点击才能完成一个简单的搜索
   - 搜索按钮不是必须的（可以自动搜索）
   - 流程不符合现代搜索体验（用户期望输入即搜索）

 2. **语言识别不清**
    - 相同书写系统的语言难以区分（如 `zh-CN` 和 `zh-TW`）
    - 搜索结果中语言信息不够醒目
    - 无法快速判断搜索结果的语言类型

## 设计目标

1. **简化操作流程**：减少点击次数，实现输入即搜索
2. **清晰的语言识别**：在搜索结果中醒目显示语言代码和地区信息
3. **清晰的操作选项**：明确区分"关联"和"创建新词句"两种操作
4. **提升效率**：一键完成关联或创建操作

## 新交互设计

### 1. 智能搜索组件

#### 组件结构

```
┌─────────────────────────────────────────┐
│  🔍 搜索词句（自动搜索）              │
│  ┌─────────────────────────────┐      │
│  │ [输入框]                    │     │
│  └─────────────────────────────┘      │
│                                      │
│  搜索结果                             │
│  ┌─────────────────────────────┐      │
│  │ [+ 创建新词句："你好"]        │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 词句文本                     │     │
│  │ [zh-CN] 中国                │     │
│  │ [关联]                     │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 词句文本                     │     │
│  │ [zh-TW] 台灣                │     │
│  │ [关联]                     │     │
│  └─────────────────────────────┘      │
└─────────────────────────────────────────┘
```

#### 功能特性

**实时搜索**
- 用户输入时自动触发搜索（带 500ms 防抖）
- 无需点击搜索按钮
- 显示搜索状态（加载中、搜索完成）

**始终显示创建新词句选项**
- 搜索结果的第一个选项始终是"创建新词句：[输入的文本]"按钮
- 无论有没有搜索结果，这个选项始终存在
- 点击直接打开创建弹窗，预填充搜索文本

**清晰的语言标记**
- 在现有搜索结果中使用语言标签（如 `[zh-CN]`、`[en-US]`）显示语言代码
- 显示地区信息（如"中国"、"台灣"）帮助区分相同书写系统的不同地区
- 醒目的视觉样式，便于快速识别
- 每个现有搜索结果只显示"关联"按钮（因为创建操作已经在第一个选项提供）

**一键操作**
- 搜索结果中每个词句提供两个操作：
  - **关联**：将当前词句关联到该词句的某个语义组
  - **创建新词句**：使用搜索文本创建新词句
- 点击后直接执行操作，无需额外确认

**快速创建**
- 当搜索结果为空时，显示醒目的"创建新词句"按钮
- 点击直接打开创建弹窗，预填充搜索文本
- 即使有搜索结果，也提供"创建新词句"选项

### 2. 语义组选择弹窗（优化版）

当用户点击"关联"时：

```
┌─────────────────────────────────────────┐
│  关联词句到语义组                     │
│  ┌─────────────────────────────┐      │
│  │ 目标词句："你好"            │     │
│  │ 语言：zh-CN • 中国          │     │
│  └─────────────────────────────┘      │
│                                      │
│  该词句已属于以下语义组：             │
│  ┌─────────────────────────────┐      │
│  │ #1 你好 / hello / こんにちは│     │
│  │     [选择此语义组]          │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ #2 问候语                 │     │
│  │     [选择此语义组]          │     │
│  └─────────────────────────────┘      │
│                                      │
│  [取消] [+ 创建新语义组]              │
└─────────────────────────────────────────┘
```

**优化点：**
- 清晰显示目标词句信息
- 列出该词句的所有语义组（包括成员）
- 一键选择语义组进行关联
- 提供创建新语义组的选项

### 3. 创建新词句弹窗（增强版）

当用户点击"创建新词句"时：

```
┌─────────────────────────────────────────┐
│  创建新词句                           │

│  │ 词句文本                     │     │
│  │ [你好]                      │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 语言      [+ 添加语言]       │     │
│  │ [zh-CN (简体中文) ▼]       │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 地区      [📍] [🗺️]       │     │
│  │ [中国]                     │     │
│  └─────────────────────────────┘      │
│                                      │
│  语义组关联                           │
│  ☑ 关联到当前语义组（如果已选择）    │

│                                      │
│  [创建] [取消]                       │
└─────────────────────────────────────────┘
```

**新增选项：**
- 可选择是否自动关联到当前语义组
- 可选择创建新语义组

## 前端实现

### 组件拆分

#### 1. SmartSearch.vue（新建）

智能搜索组件，封装搜索逻辑和 UI。

```vue
<template>
  <div class="smart-search-container">
    <!-- 搜索输入区 -->
    <div class="search-input-area">
      <input
        v-model="searchQuery"
        :placeholder="$t('search_placeholder')"
        class="search-input"
        @input="handleSearchInput"
      />
    </div>

    <!-- 搜索结果区（始终显示） -->
    <div v-if="searched" class="search-results">
      <!-- 第一个选项：始终显示创建新词句 -->
      <div class="create-new-option" @click="handleCreateNew(searchQuery)">
        <div class="create-new-content">
          <svg class="create-icon">+</svg>
          <span class="create-text">{{ $t('add_expression') }}: "{{ searchQuery }}"</span>
        </div>
      </div>

      <!-- 搜索状态 -->
      <div v-if="loading" class="search-status">
        <loading-spinner />
        <span>{{ $t('searching') }}</span>
      </div>

      <!-- 现有搜索结果 -->
      <div v-if="!loading && hasResults" class="existing-results">
        <div v-for="result in searchResults" :key="result.id" class="search-result-item">
          <div class="result-content">
            <div class="result-text">{{ result.text }}</div>
            <div class="result-meta">
              <span class="language-tag">[{{ result.language_code }}]</span>
              <span v-if="result.region_name" class="region-tag">
                {{ result.region_name }}
              </span>
            </div>
          </div>
          <div class="result-actions">
            <button @click="handleAssociate(result)" class="action-btn associate-btn">
              {{ $t('associate') }}
            </button>
          </div>
        </div>
      </div>

      <!-- 无搜索结果提示 -->
      <div v-if="!loading && noResults" class="no-results-hint">
        {{ $t('no_expressions_found') }}
      </div>
    </div>
  </div>
</template>
```

### 交互流程

#### 流程 1：创建新词句

1. 用户在智能搜索框输入文本（如"你好"）
2. 系统自动搜索（带防抖）
3. 搜索结果区显示，第一个选项是 `[+ 创建新词句："你好"]`
4. 用户点击第一个选项
5. 打开创建弹窗，文本字段已预填充为"你好"
6. 用户完成表单填写（语言、地区等）
7. 提交创建，可选择是否关联到语义组
8. 刷新详情页

#### 流程 2：关联现有词句

1. 用户在智能搜索框输入文本
2. 系统自动搜索（带防抖）
3. 搜索结果区显示，第一个选项始终是创建新词句
4. 下方显示搜索到的现有词句
5. 用户找到想要关联的词句，点击其"关联"按钮
6. 弹出语义组选择弹窗，显示该词句的所有语义组
7. 用户选择要关联的语义组
8. 完成关联，刷新详情页

## 后端支持

### API 设计

#### 搜索 API（已存在，需增强）

`GET /api/v1/search`

**现有参数：**
- `q`: 搜索查询词
- `include_meanings`: 是否包含语义组信息

**新增参数：**
- `exclude_id`: 排除特定词句 ID（可选，用于排除当前词句）

**示例请求：**
```http
GET /api/v1/search?q=你好&include_meanings=true&exclude_id=123
```

#### 关联 API（已存在）

`POST /api/v1/expressions/:expr_id/meanings`

**请求体：**
```json
{
  "meaning_id": 456
}
```

#### 创建词句 API（已存在）

`POST /api/v1/expressions`

**现有功能保持不变，支持通过 `initialMeaningId` 预先关联到语义组**
