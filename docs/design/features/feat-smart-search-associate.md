# 詳情頁智能搜索與關聯設計

## 概述

優化詳情頁的"搜索和關聯"功能，簡化用戶操作流程，提升用戶體驗。主要解決兩個問題：
1. 當前交互流程繁瑣（需要多次點擊、手動觸發搜索）
2. 相同書寫系統的語言間搜索幹擾（如簡體中文/繁體中文）

## 當前問題分析

### 現有交互流程

1. 用戶點擊"搜索和關聯"標籤頁或"關聯詞句"按鈕
2. 進入搜索模式，需要手動點擊啓用搜索
3. 輸入查詢詞
4. 點擊"搜索"按鈕
5. 查看搜索結果
6. 如果沒找到結果，點擊"添加詞句"按鈕

### 存在的問題

1. **操作繁瑣**
   - 需要多次點擊才能完成一個簡單的搜索
   - 搜索按鈕不是必須的（可以自動搜索）
   - 流程不符合現代搜索體驗（用戶期望輸入即搜索）

 2. **語言識別不清**
    - 相同書寫系統的語言難以區分（如 `zh-CN` 和 `zh-TW`）
    - 搜索結果中語言信息不夠醒目
    - 無法快速判斷搜索結果的語言類型

## 設計目標

1. **簡化操作流程**：減少點擊次數，實現輸入即搜索
2. **清晰的語言識別**：在搜索結果中醒目顯示語言代碼和地區信息
3. **清晰的操作選項**：明確區分"關聯"和"創建新詞句"兩種操作
4. **提升效率**：一鍵完成關聯或創建操作

## 新交互設計

### 1. 智能搜索組件

#### 組件結構

```
┌─────────────────────────────────────────┐
│  🔍 搜索詞句（自動搜索）              │
│  ┌─────────────────────────────┐      │
│  │ [輸入框]                    │     │
│  └─────────────────────────────┘      │
│                                      │
│  搜索結果                             │
│  ┌─────────────────────────────┐      │
│  │ [+ 創建新詞句："你好"]        │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 詞句文本                     │     │
│  │ [zh-CN] 中國                │     │
│  │ [關聯]                     │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 詞句文本                     │     │
│  │ [zh-TW] 臺灣                │     │
│  │ [關聯]                     │     │
│  └─────────────────────────────┘      │
└─────────────────────────────────────────┘
```

#### 功能特性

**實時搜索**
- 用戶輸入時自動觸發搜索（帶 500ms 防抖）
- 無需點擊搜索按鈕
- 顯示搜索狀態（加載中、搜索完成）

**始終顯示創建新詞句選項**
- 搜索結果的第一個選項始終是"創建新詞句：[輸入的文本]"按鈕
- 無論有沒有搜索結果，這個選項始終存在
- 點擊直接打開創建彈窗，預填充搜索文本

**清晰的語言標記**
- 在現有搜索結果中使用語言標籤（如 `[zh-CN]`、`[en-US]`）顯示語言代碼
- 顯示地區信息（如"中國"、"臺灣"）幫助區分相同書寫系統的不同地區
- 醒目的視覺樣式，便於快速識別
- 每個現有搜索結果只顯示"關聯"按鈕（因爲創建操作已經在第一個選項提供）

**一鍵操作**
- 搜索結果中每個詞句提供兩個操作：
  - **關聯**：將當前詞句關聯到該詞句的某個語義組
  - **創建新詞句**：使用搜索文本創建新詞句
- 點擊後直接執行操作，無需額外確認

**快速創建**
- 當搜索結果爲空時，顯示醒目的"創建新詞句"按鈕
- 點擊直接打開創建彈窗，預填充搜索文本
- 即使有搜索結果，也提供"創建新詞句"選項

### 2. 語義組選擇彈窗（優化版）

當用戶點擊"關聯"時：

```
┌─────────────────────────────────────────┐
│  關聯詞句到語義組                     │
│  ┌─────────────────────────────┐      │
│  │ 目標詞句："你好"            │     │
│  │ 語言：zh-CN • 中國          │     │
│  └─────────────────────────────┘      │
│                                      │
│  該詞句已屬於以下語義組：             │
│  ┌─────────────────────────────┐      │
│  │ #1 你好 / hello / こんにちは│     │
│  │     [選擇此語義組]          │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ #2 問候語                 │     │
│  │     [選擇此語義組]          │     │
│  └─────────────────────────────┘      │
│                                      │
│  [取消] [+ 創建新語義組]              │
└─────────────────────────────────────────┘
```

**優化點：**
- 清晰顯示目標詞句信息
- 列出該詞句的所有語義組（包括成員）
- 一鍵選擇語義組進行關聯
- 提供創建新語義組的選項

### 3. 創建新詞句彈窗（增強版）

當用戶點擊"創建新詞句"時：

```
┌─────────────────────────────────────────┐
│  創建新詞句                           │
│  ┌─────────────────────────────┐      │
│  │ 詞句文本                     │     │
│  │ [你好]                      │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 語言      [+ 添加語言]       │     │
│  │ [zh-CN (簡體中文) ▼]       │     │
│  └─────────────────────────────┘      │
│  ┌─────────────────────────────┐      │
│  │ 地區      [📍] [🗺️]       │     │
│  │ [中國]                     │     │
│  └─────────────────────────────┘      │
│                                      │
│  語義組關聯                           │
│  ☑ 關聯到當前語義組（如果已選擇）    │
│  ☐ 創建新語義組                      │
│                                      │
│  [創建] [取消]                       │
└─────────────────────────────────────────┘
```

**新增選項：**
- 可選擇是否自動關聯到當前語義組
- 可選擇創建新語義組

## 前端實現

### 組件拆分

#### 1. SmartSearch.vue（新建）

智能搜索組件，封裝搜索邏輯和 UI。

```vue
<template>
  <div class="smart-search-container">
    <!-- 搜索輸入區 -->
    <div class="search-input-area">
      <input
        v-model="searchQuery"
        :placeholder="$t('search_placeholder')"
        class="search-input"
        @input="handleSearchInput"
      />
    </div>

    <!-- 搜索結果區（始終顯示） -->
    <div v-if="searched" class="search-results">
      <!-- 第一個選項：始終顯示創建新詞句 -->
      <div class="create-new-option" @click="handleCreateNew(searchQuery)">
        <div class="create-new-content">
          <svg class="create-icon">+</svg>
          <span class="create-text">{{ $t('add_expression') }}: "{{ searchQuery }}"</span>
        </div>
      </div>

      <!-- 搜索狀態 -->
      <div v-if="loading" class="search-status">
        <loading-spinner />
        <span>{{ $t('searching') }}</span>
      </div>

      <!-- 現有搜索結果 -->
      <div v-if="!loading && hasResults" class="existing-results">
        <div v-for="result in searchResults" :key="result.id" class="search-result-item">
          <div class="result-content">
            <div class="result-text">{{ result.text }}</div>
            <div class="result-meta">
              <span class="language-tag">[{{ result.language_code }}]</span>
              <span v-if="result.region_name" class="region-tag">
                {{ result.region_name }}
              </span>
            </div>
          </div>
          <div class="result-actions">
            <button @click="handleAssociate(result)" class="action-btn associate-btn">
              {{ $t('associate') }}
            </button>
          </div>
        </div>
      </div>

      <!-- 無搜索結果提示 -->
      <div v-if="!loading && noResults" class="no-results-hint">
        {{ $t('no_expressions_found') }}
      </div>
    </div>
  </div>
</template>
```

#### 2. MeaningSelectionModal.vue（新建/優化）

語義組選擇彈窗，用於關聯操作。

#### 3. CreateExpression.vue（增強）

增強創建彈窗，添加語義組關聯選項。

### 交互流程

#### 流程 1：創建新詞句

1. 用戶在智能搜索框輸入文本（如"你好"）
2. 系統自動搜索（帶防抖）
3. 搜索結果區顯示，第一個選項是 `[+ 創建新詞句："你好"]`
4. 用戶點擊第一個選項
5. 打開創建彈窗，文本字段已預填充爲"你好"
6. 用戶完成表單填寫（語言、地區等）
7. 提交創建，可選擇是否關聯到語義組
8. 刷新詳情頁

#### 流程 2：關聯現有詞句

1. 用戶在智能搜索框輸入文本
2. 系統自動搜索（帶防抖）
3. 搜索結果區顯示，第一個選項始終是創建新詞句
4. 下方顯示搜索到的現有詞句
5. 用戶找到想要關聯的詞句，點擊其"關聯"按鈕
6. 彈出語義組選擇彈窗，顯示該詞句的所有語義組
7. 用戶選擇要關聯的語義組
8. 完成關聯，刷新詳情頁

## 後端支持

### API 設計

#### 搜索 API（已存在，需增強）

`GET /api/v1/search`

**現有參數：**
- `q`: 搜索查詢詞
- `include_meanings`: 是否包含語義組信息

**新增參數：**
- `exclude_id`: 排除特定詞句 ID（可選，用於排除當前詞句）

**示例請求：**
```http
GET /api/v1/search?q=你好&include_meanings=true&exclude_id=123
```

#### 關聯 API（已存在）

`POST /api/v1/expressions/:expr_id/meanings`

**請求體：**
```json
{
  "meaning_id": 456
}
```

#### 創建詞句 API（已存在）

`POST /api/v1/expressions`

**現有功能保持不變，支持通過 `initialMeaningId` 預先關聯到語義組**
